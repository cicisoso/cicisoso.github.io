Introduction

MQL5指南

MetaQuotes Language 5 (MQL5) is a high-level language designed for developing technical indicators, trading robots and utility applications, which automate financial trading. MQL5 has been developed by MetaQuotes Software Corp. for their trading platform. The language syntax is very close to C++ enabling programmers to develop applications in the object-oriented programming (OOP) style.

MQL5是一个面向对象的高级语言，设计用于开发技术指标，自动交易机器人和工具应用，从而实现自动化金融交易。 MQL5是由MetaQuotes Software Corp. 为其交易平台开发。这个语言的语法和c++很相似，开发人员可以用面向对象的风格开发程序。

In addition to the MQL5 language, the trading platform package also includes the MetaEditor IDE with highly advanced code writing tools, such as templates, snippets, debugging, profiling and auto completion tools, as well as built-in MQL5 Storage enabling file versioning.

除了MQL5语言，交易平台还包括了高级的编写工具 MetaEditor IDE，例如模板，代码，debug，profilling和自动补全工具，还有内置的MQL5 Storage 提供文件版本记录。

The language support is available on the MQL5.community website, which contains a huge free CodeBase and a plethora of articles. These articles cover all the aspects of the modern trading, including neural networks, statistics and analysis, high-frequency trading, , testing and optimization of trading strategies, use of trading automation robots, and more.

MQL5社区网站有语言帮助，包含了大量的免费代码和海量的文章。这些文章包括目前交易的方方面面，包括神经网络，统计和分析，高频交易，套利，测试和优化交易策略，自动交易机器人等等。

Traders and MQL5 program developers can communicate on the forum, order and develop applications using the Freelance service, as well as buy and sell protected programs in the Market of automated trading applications.

交易员和MQL5程序开发者可以通过社区论坛进行交流，通过Freelance来下单和开发应用，同样还可以在Market买或者卖掉私有的自动交易程序。

The MQL5 language provides specialized trading functions and predefined event handlers to help programmers develop Expert Advisors (EAs), which automatically control trading processes following specific trading rules. In addition to EAs, MQL5 allows developing custom technical indicators, scripts and libraries.

MQL5语言提供了专门的交易函数（trading functions）和预定义的事件处理函数来帮助开发者开发EA程序，从而自动控制交易流程符合特定交易规则。除了EA外，MQL5允许开发自定义的技术指标technical indicators，脚本和库。

This MQL5 language reference contains functions, operations, reserved words and other language constructions divided into categories. The reference also provides descriptions of Standard Library classes used for developing trading strategies, control panels, custom graphics and enabling file access.

MQL5指南根据函数、运行，保留关键字和其他语言构造划分成几个类目。这份指南同样提供标准库Standard Library 类的介绍，用来开发交易策略、控制面板、自定义图表和提供文件访问。

Additionally, the CodeBase contains the ALGLIB numerical analysis library, which can be used for solving various mathematical problems.

另外，CodeBase包含的ALGLIB数值分析库，可以用来解决多种数学问题。

Types of MQL5 Applications

MQL5 programs are divided into four specialized types based on the trading automation tasks that they implement:

MQL5根据交易自动化实现方式分为四个特殊的类型：

Expert Advisor is an automated trading system linked to a chart. An Expert Advisor contains event handlers to manage predefined events which activate execution of appropriate trading strategy elements. For example, an event of program initialization and deinitializtion, new ticks, timer events, changes in the Depth of Market, chart and custom events.

EA是一个绑定到图表的自动化交易系统。一个EA包含事件处理器，用于实现预定义事件，这些事件在合适的交易策略要素下触发。例如，程序初始化和销毁的时间，新tick，定时器事件，市场深度变化事件，图和自定义事件。

In addition to calculating trading signals based on the implemented rules, Expert Advisors can also automatically execute trades and send them directly to a trading server. Expert Advisors are stored in \MQL5\Experts.

除了计算交易信号和实现规则，EA还能够自动的执行交易并直接发送到交易服务器。EA保存在 \MQL5\Experts。

Custom Indicator*s is a technical indicator developed by a user in addition to standard indicators integrated into the trading platform. Custom indicators, as well as standard ones, cannot trade automatically, but only implement analytical functions. Custom indicators can utilize values of other indicators for calculations, and can be called from Expert Advisors. Custom indicators are stored in \MQL5\Indicators*.

自定义指标是一个交易者开发的技术指标，作为交易平台整合的标准指标的补充。自定义指标，或者标准指标，都不能自动交易，只能实现分析功能。自定义指标可以利用其它指标的值进行计算，可以被EA调用。自定义指标保存在\MQL5\Indicators

Script is a program for a single execution of an action. Unlike Expert Advisors, scripts do not handle any event except for trigger. A script code must contain the OnStart handler function.

脚本是一个只执行一次的动作的程序。和EA不同，脚本不处理任何事件除了触发。一个脚本代码必须包含OnStart事件。

Service is a program that, unlike indicators, Expert Advisors and scripts, does not require to be bound to a chart to work. Like scripts, services do not handle any event except for trigger. To launch a service, its code should contain the OnStart handler function. Services do not accept any other events except Start, but they are able to send custom events to charts using EventChartCustom. Services are stored in \MQL5\Services.

服务不同于指标、EA和脚本，不需要绑定到图标中运行。像脚本，服务不需要处理任何事件除了触发自己。启动一个服务，这个服务应包含OnStart处理函数。服务不能接受除了Start外的任何事件，但是服务能够通过EventChartCustom发送自定义的时间到图标。服务存放在\MQL5\Services。

Library is a set of custom functions. Libraries are intended to store and distribute commonly used algorithms of custom programs. Libraries are stored in \MQL5\Libraries.

库是一个自定义功能的集合。库用于存放和分类常用的自定义程序中的算法。库保存在\MQL5\Libraries。

Include File is a source text of the most frequently used blocks of custom programs. Such files can be included into the source texts of Expert Advisors, scripts, custom indicators, and libraries at the compiling stage. The use of included files is more preferable than the use of libraries because of additional burden occurring at calling library functions.

包含文件是自定义程序最常使用的源文本。这些文件可以被EA、脚本，自定义指标和库文件的源文本在编译时引用。使用包含文件比使用库更好，因为调用库文件功能会有额外的开销。
